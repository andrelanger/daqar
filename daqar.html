DaQAR - The Data Quality Assessment Requirements Vocabulary

(author names currently anonymized due to ongoing double-blind conference review)

Abstract

The World Wide Web represents a tremendous source of information with resources of varying data quality from almost arbitrary knowledge domains. The decision process to select the best data source for current business requirements is not trivial. In the past, research has already focused on related quality aspects such as vocabularies to represent data quality metrics and measurements (DQV) or notations to represent and validate structural requirements (SHACL). But a consistent universal semantic approach to define specific quality requirements for assessment purposes from the data consumer side is still missing. Therefore, we address this challenge and present DaQAR - an ontology that is capable of defining arbitrary quality requirements on both data instance, schema and service level in a uniform fashion. This formalization includes desired measurement result boundaries, a customizable calculation as well as comprehensible quality rating scores. It can be used for data quality assessment purposes to compare multiple eligible data resources on particular metrics and attributes of current interest.

2. Vocabulary Overview

DaQAR seeks to reuse already established data related vocabularies, primarily DCAT and DQV. It includes the following concepts depicted in figure 1. A machine-readable version of this ontology is available at daqar.owl or daqar.ttl.

3.Vocabulary specification

3.1 Class: QualityRequirementSpec

RDF Class: dqr:QualityRequirementSpec
Definition: Represents a specification of requirements on a data set or data service from consumer side for quality assessment purposes
Note: A dqr:QualityRequirementSpec is an abstract definition of requirements, conditions and desired metric values. It is basically independent from a concret data set and can be used among multiple data sets or data set states for comparison purposes.

The following properties can be used with instances of this class:

3.1.1 Property: hasRequirements

RDF Property: dqr:hasRequirements
Definition: Indicates the QualityRequirements that have to be checked
Domain: dqr:QualityRequirementSpec
Range: dqr:QualityRequirement
Minimum cardinality:1
Note: All QualityRequirements should be computeable independently.

3.1.2 Property: hasAssessment

RDF Property: dqr:hasAssessment
Definition: Indicates the definition of an overall assessment relation how the quality of a data source has to be calculated
Domain: dqr:QualityRequirementSpec
Range: dqr:QualityAssessmentCombination
Maximum cardinality:1
Note: If this property is not provided, the dqr:QualityRequirementSpec represents a list of quality metrics without an assessment instruction.

3.2 Class: QualityAssessmentCombination

RDF Class: dqr:QualityAssessmentCombination
Definition: Represents an instruction on how to combine the results from multiple dqr:QualityAssessments to an overall assessment score
Note: If there is only one quality requirement that has to be assessed, this class represents simply a proxy container for this calculation.

The following properties can be used with instances of this class:

3.2.1 Property: hasReturnType

RDF Property: dqr:hasReturnType
Definition: Indicates the numeric type of the returned assessment result
Domain: dqr:QualityAssessmentCombination
Range: dqr:ReturnType
Maximum cardinality:1
Note: The class dqr:ReturnType currently contains the concepts
  • dqr:QualityRatio for a decimal representation 0.0 (worst) <= DQ <= 1.0 (best)
  • dqr:QualityStar for an integer representation 0 (worst) <= DQ <= 5 (best)
  • dqr:QualityGrade for an integer representation 1 (best) <= DQ <= 6 (worst)
If the property is not provided at all, dqr:QualityRatio is assumed.

3.2.2 Property: hasAssessmentTerms

RDF Property: dqr:hasAssessmentTerms
Definition: Represents a list of independent QualityAssessment subcalculations that should be combined to an overall assessment result
Domain: dqr:QualityAssessmentCombination
Range: dqr:QualityAssessmentTerm
Minimum cardinality:1

3.3 Class: QualityAssessmentTerm

RDF Class: dqr:QualityAssessmentTerm
Definition: Represents a weighted assessment sub-calculation for a particular requirement

The following properties can be used with instances of this class:

3.3.1 Property: hasWeight

RDF Property: dqr:hasWeight
Definition: Indicates the influence of the current assessment to the overall assessment score
Domain: dqr:QualityAssessmentTerm
Range: xsd:decimal, 0.0 <= value <= 1.0
Maximum cardinality:1
Note: dqr:QualityAssessmentCombination basically calculates a normalized value, so the sum of all weights should be 1.0 in total if not otherwise explicitly intended

3.3.2 Property: hasAssessment

RDF Property: dqr:hasAssessment
Definition: Indicates a quality assessment that shall be calculated
Domain: dqr:QualityAssessmentTerm
Range: dqr:QualityAssessment
Minimum cardinality:1
Maximum cardinality:1

3.4 Class: QualityRequirement

RDF Class: dqr:QualityRequirement
Definition: Represents a particular measurable quality aspect of interest whose result is assesseable

The following properties can be used with instances of this class:

3.4.1 Property: hasRequirementType

RDF Property: dqr:hasRequirementType
Definition: Indicates the type of examined quality aspect
Domain: dqr:QualityRequirement
Range: dqr:RequirementType
Maximum cardinality:1
Note: Basically, any URI-representable concept can be used as a requirement specification. URIs of particular quality metrics are of major interest. But we could also use SHACL URIs from template graphs as a requirement description. In order to programmatically distinguish these different concepts, the class dqr:RequirementType currently contains the concepts
  • dqr:SimpleMetric for directly calculable metrics such as DQV, daq or ISO metrics
  • dqr:ShapeConstraint for SHACL-like template graph descriptors that are run by a separate processor
The dqr:RequirementType class is open for future extensions.
If the property is not mentioned at all, dqr:SimpleMetric is assumed.

3.4.2 Property: hasRequirementConcept

RDF Property: dqr:hasRequirementConcept
Definition: Represents the particular quality concepts in form of a URI
Domain: dqr:QualityRequirement
Minimum cardinality:1
Maximum cardinality:1
Note: This can be a daq:Metric, a dqv:Metric, a SHACL URL or any other URI representing a quality requirement

3.4.3 Property: hasImplementation

RDF Property: dqr:hasImplementation
Definition: Indicates if the calculation relies on a concrete, mandatory implementation of a quality requirement concept
Domain: dqr:QualityRequirement
Maximum cardinality:1
Note: Sometimes, the same quality metric is implemented in different interpretations and its calculation can lead to different results. This property allows it to specify an optional URI representing a concrete tool, service or implementation that should be used for measuring the concept defined

3.4.4 Property: expectedDataType

RDF Property: dqr:expectedDataType
Definition: Indicates the expected return type of the measurement result
Domain: dqr:QualityRequirement
Range: xsd:anySimpleType
Maximum cardinality:1
Note: Currently, some descriptions of recommended DQ metrics leave it open, e.g., if the caculation should return an integer value for all instances satisfying the requirement, or a ratio, or something totally different. The dqr:expectedDataType makes this definition explicit.

3.4.5 Property: hasUnit

RDF Property: dqr:hasUnit
Definition: Indicates the unit of the measurement result
Domain: dqr:QualityRequirement
Maximum cardinality:1
Note: It is helpful to explicitly state the unit of a measurement result in the requirement description if applicable, e.g., if it returns seconds or milliseconds. We currently reommend to use the Measurement Units Ontology (MUO).

3.4.6 Property: hasAssessment

RDF Property: dqr:hasAssessment
Definition: Indicates a calculation function for assessing the measurement result
Domain: dqr:QualityRequirement
Range: dqr:QualityAssessment
Maximum cardinality:1
Note: The object can either be an individual definition of an assessment function. It can also be a URL to a well-known assessment function that was already defined somewehere else. If the property is not provided at all, the quality requirement description simply states the desired quality concept of interest along with some additional requirement meta data.

3.4.7 Property: hasAssessmentParam

RDF Property: dqr:hasAssessmentParam
Definition: Specifies parameter values that can be used in combination with well-known assessment function definitions.
Domain: dqr:QualityRequirement
Range: dqr:AssessmentParam

3.5 Class: AssessmentParam

RDF Class: dqr:AssessmentParam
Definition: Represents a parameter with a defined value that can be reused by an assessment tool in combination with a well-known dqr:Assessment function
Note: In order to map and use these parameter values, they should have a well-known parameter type that corresponds to the parameter type used in the well-known assessment function

The following properties can be used with instances of this class:

3.5.1 Property: hasParamType

RDF Property: dqr:hasParamType
Definition: Indicates the type of the assessment param
Domain: dqr:AssessmentParam
Range: dqr:ParamType
Minimum cardinality:1
Maximum cardinality:1
Note: The class dqr:Param is open for extensions and currently contains the concepts
  • dqr:LowerThanParam
  • dqr:LowerEqualThanParam
  • dqr:AverageParam
  • dqr:GreaterThanParam
  • dqr:GreaterEqualThanParam

3.5.2 Property: hasValue

RDF Property: dqr:hasValue
Definition: Specified the value of the assessment parameter
Domain: dqr:AssessmentParam
Range: xsd:anySimpleType
Minimum cardinality:1
Maxmium cardinality:1

3.6 Class: QualityAssessment

RDF Class: dqr:QualityAssessment
Definition: Represents the defintion of a function of measurement boundaries on how to assess the measured value of a related quality concept of interest
Note: This can either be an individually defined assessment function or a well-known assessment function. A well-known assessment function is a globally provided QualityAssessment URL for re-occuring assessment function patterns.
Currently, the following well-known QualityAssessment function descriptions are supported:
  • dqr:MaxValueAssessmentFunction - a linear quality assessment function for a measurement value from 0.0 (best) to a max value (worst), dqr:AssessmentParam dqr:LowerThanParam
This list is open for future extensions.

The following properties can be used with instances of this class:

3.6.1 Property: hasReturnType

RDF Property: dqr:hasReturnType
Definition: Indicates the type of the returned value for the assessment result
Domain: dqr:QualityAssessment
Range: dqr:ReturnType
Maximum cardinality:1
Note: See section 3.2.1 for further details

3.6.2 Property: hasReturnValue

RDF Property: dqr:hasReturnValue
Definition: Indicates the computation type of the returned assessment value
Domain: dqr:QualityAssessment
Range: dqr:ReturnValue
Maxmium cardinality:1
Note: This is an additional meta information, if the assessment value is actually calculated by a particular assessment function. The class dqr:ReturnValue currently contains the concepts
  • dqr:FunctionalValue
  • dqr:ConstantValue
and is open for future extensions.

3.6.3 Property: hasCondition

RDF Property: dqr:hasCondition
Definition: Specifies an operation or a condition that has to be met by the measured value in order to calculate an assessment result
Domain: dqr:QualityAssessment
Range: dqr:AssessmentCondition
Minimum cardinality:1

3.7 Class: AssessmentCondition

RDF Class: dqr:AssessmentCondition
Definition: Represents a measurement value range and a mapping function to an assessment value

The following properties can be used with instances of this class:

3.7.1 Property: hasValueLowerThan

RDF Property: dqr:hasValueLowerThan
Definition: Indicates what happens if the measured value is lower than the specified value
Domain: dqr:AssessmentCondition
Range: xsd:anySimpleType
Maximum cardinality:1

3.7.2 Property: hasValueLowerEqualThan

RDF Property: dqr:hasValueLowerEqualThan
Definition: Indicates what happens if the measured value is lower or euqal than the specified value
Domain: dqr:AssessmentCondition
Range: xsd:anySimpleType
Maximum cardinality:1

3.7.3 Property: hasValueGreaterThan

RDF Property: dqr:hasValueGreaterThan
Definition: Indicates what happens if the measured value is greater than the specified value
Domain: dqr:AssessmentCondition
Range: xsd:anySimpleType
Maximum cardinality:1

3.7.4 Property: hasValueGreaterEqualThan

RDF Property: dqr:hasValueGreaterEqualThan
Definition: Indicates what happens if the measured value is greater or euqal than the specified value
Domain: dqr:AssessmentCondition
Range: xsd:anySimpleType
Maximum cardinality:1

3.7.5 Property: hasAssessmentFunction

RDF Property: dqr:hasAssessmentFunction
Definition: Defines a mapping function how to represent the measured value as an assessment value
Domain: dqr:AssessmentCondition
Range: dqr:AssessmentFunction
Minimum cardinality:1
Maximum cardinality:1

3.8 Class: AssessmentFunction

RDF Class: dqr:AssessmentFunction
Definition: Represents a mapping function to map an actual measurement value to a corrresponding assessment value

The following properties can be used with instances of this class:

3.8.1 Property: hasFunctionType

RDF Property: dqr:hasFunctionType
Definition: Indicates the functional type of the calculation
Domain: dqr:AssessmentFunction
Range: dqr:FunctionType
Minimum cardinality:1
Maximum cardinality:1
Note: This property specifies the mathematical function how to map a measured value to an assessment value. The class dqr:FunctionType currently contains the concepts
  • dqr:LinearFunction - defined through a dqr:hasStartValue and dqr:hasEndValue
  • dqr:ConstantFunction - defined through a dqr:hasReturnValue
  • dqr:CustomFunction - defined through a dqr:hasCustomFunctionTerm
and is open for future extensions. It can also be considered to represent this concept through other appropriate existing vocabularies.

3.8.2 Property: hasFunctionParam

RDF Property: dqr:hasFunctionParam
Definition: Defines values for the functional parameters required by dqr:FunctionType
Domain: dqr:AssessmentFunction
Range: xsd:anySimpleType
Note: The property name of the provided dqr:FunctionParam s has to correspond to the params required by a dqr:FunctionType function

3.9 Class: FunctionParam

RDF Class: dqr:FunctionParam
Definition: Represents a bound value used by a dqr:FunctionType function for calculation purposes

The following properties can be used with instances of this class:

3.9.1 Property: hasStartValue

RDF Property: dqr:hasStartValue
Definition: Represents the start value of a function
Domain: dqr:FunctionParam
Range: xsd:anySimpleType
Maximum cardinality:1

3.9.2 Property: hasEndValue

RDF Property: dqr:hasEndValue
Definition: Represents the end value of a function
Domain: dqr:FunctionParam
Range: xsd:anySimpleType
Maximum cardinality:1

3.9.3 Property: hasReturnValue

RDF Property: dqr:hasReturnValue
Definition: Represents the (constant) return value of a function
Domain: dqr:FunctionParam
Range: xsd:anySimpleType
Maximum cardinality:1

4. Example Usage

This section shows some examples to illustrate the application of the Data Quality Assessment Requirements vocabulary.

4.1 Expressing data quality requirements

:OverallRequirement
	a dqr:QualityRequirementSpec ;
	dqr:hasRequirements :requirement1, :requirement2 ;
	dqr:hasAssessment :OverallAssessment
	.

4.2 Expressing a particular requirement

:requirement1
	a dqr:QualityRequirement ;
	dqr:hasRequirementType dqr:SimpleMetric ;
	dqr:hasRequirementConcept dqv:LowLatencyMetric ;
	dqr:hasImplementation ex:LowLatencyImpl ;
	dqr:expectedDataType xsd:double ;
	dqr:hasUnit muo:Seconds 
	dqr:hasAssessment :assessment1
	dqr:hasAssessmentParam [
		a dqr:AssessmentParam ;
		dqr:hasParamType dqr:LowerThanParam ;
		dqr:hasValue "2.0"^^xsd:double 
	]
	.

:requirement2
	a dqr:QualityRequirement ;
	dqr:hasRequirementType dqr:ShapeConstraint ;
	dqr:hasRequirementConcept shacl:MyPersonPropertyShape ;
	dqr:expectedDataType xsd:integer ;
	dqr:hasAssessment :assessment2
	dqr:hasAssessmentParam [
		a dqr:AssessmentParam ;
		dqr:hasParamType dqr:LowerThanParam ;
		dqr:hasValue "100"^^xsd:integer 
	]
	.

4.3 Expressing an assessment function

:assessment1
	a dqr:QualityAssessment ;
	dqr:hasReturnType dqr:QualityRatio ;
	dqr:hasReturnValue dqr:FunctionalValue ;
	dqr:hasCondition [
		a dqr:AssessmentCondition ;
		dqr:hasValueLowerThan "0.0"^^xsd:double ;
		dqr:hasAssessmentFunction [
			a dqr:AssessmentFunction ;
			dqr:hasFunctionType dqr:ConstantFunction ;
			dqr:hasFunctionParam [
				a dqr:FunctionParam ;
				dqr:hasReturnValue "0.0"^^xsd:double
			]
		]
	]
	dqr:hasCondition [
		a dqr:AssessmentCondition ;
		dqr:hasValueGreaterEqualThan "0.0"^^xsd:double ;
		dqr:hasValueLowerEqualThan "2.0"^^xsd:double ;
		dqr:hasAssessmentFunction [
			a dqr:AssessmentFunction ;
			dqr:hasFunctionType dqr:LinearFunction ;
			dqr:hasFunctionParam [
				a dqr:FunctionParam ;
				dqr:hasStartValue "1.0"^^xsd:double
			], [
				a dqr:FunctionParam ;
				dqr:hasEndValue "0.0"^^xsd:double
			]
		]
	]		
	dqr:hasCondition [
		a dqr:AssessmentCondition ;
		dqr:hasValueGreaterThan "2.0"^^xsd:double ;
		dqr:hasAssessmentFunction [
			a dqr:AssessmentFunction ;
			dqr:hasFunctionType dqr:ConstantFunction ;
			dqr:hasFunctionParam [
				a dqr:FunctionParam ;
				dqr:hasReturnValue "0.0"^^xsd:double 
			]
		]
	]		
	.

4.4 Instantiating a particular dataset 

:dataset1 
    a dcat:Dataset ;
    dcterms:title "My dataset" ; 
    dcat:distribution :distribution1
    .

:distribution1
    a dcat:Distribution ;
    dcterms:title "SPARQL endpoint with RDF/XML distribution of dataset" ;
    dqv:hasQualityMeasurement :measurement1, :measurement2
    .

4.4 Measuring a quality metric via DQV 

:measurement1 
    a dqv:QualityMeasurement ;
    dqv:computedOn :distribution1 ;
    dqv:isMeasurementOf :LowLatencyMetric ;
    dqv:value "0.8627876"^^xsd:double 
    .

4.5 Calculating an overall assessment score 

:OverallAssessment	
	a dqr:QualityAssessmentCombination ;
	dqr:hasReturnType dqr:QualityRatio ;
	dqr:hasAssessmentTerms [
		a dqr:QualityAssessmentTerm ;
		dqr:hasWeight "0.6"xsd:double ;
		dqr:hasAssessment :assessment1 
	], [
		a dqr:QualityAssessmentTerm ;
		dqr:hasWeight "0.4"^^xsd:double ;
		dqr:hasAssessment :assessment2
	]
	.
